### Muon+ Row-wise Normalization

After Newton-Schulz5 orthogonalization, the update matrix is row-normalized (`F.normalize(update, dim=1)`) before the parameter update. This is the "Muon+" variant from [arxiv:2602.21545](https://arxiv.org/abs/2602.21545). Not present in the base submission.

### Legal Score-First TTT

Each chunk of the validation set is scored under `torch.no_grad()` first, then the model trains on those already-scored tokens. No data leakage. 30 chunks × ~970K windows, 3 epochs per chunk. Reduces BPB by ~0.024 nats over the quantized baseline.

### Learnings from running autoresearch with arc

**Isolation and reproducibility are non-negotiable.** Each experiment lives in its own git worktree with a dedicated commit before launch (`arc submit` auto-commits). One commit = one experiment. Without this, logs are ambiguous and results can't be reproduced or compared cleanly.

**Enforce experiment structure.** arc enforces a consistent layout — submission scripts, run logs, hypothesis notes (`arc hyp`), and verdicts (`arc result`) — so that the experiment tree stays navigable as it grows. Ad-hoc logging quickly becomes unreadable at scale.

**Getting the optimization objective right is the most important step.** Models are very good at overfitting to whatever objective they're given. If the objective is ill-defined or misaligned with the actual goal (e.g. BPB on a fixed val set vs. final leaderboard score), autoresearch outcomes are meaningless regardless of how many experiments are run. Defining and validating the objective is the first thing to get right.

**Single-agent multi-execution beats multi-agent single-execution.** Running one agent that launches multiple parallel training jobs is more reliable and cheaper to coordinate than running multiple independent agents. The agent maintains full context over the experiment tree and can route results into a coherent hypothesis board.

**But single-agent explores depth, not breadth.** A single agent will go deep into one direction (e.g. int6 compression stack) and may miss orthogonal ideas entirely. The right structure is multiple agents each owning a different research direction, each with multi-execution capability — agents explore horizontally, execution parallelizes vertically within each direction.

This is an preview of an autoresearch process in-progress.
